## Описание проекта

Клиенту (РЖД) требуется веб-интерфейс для удобной работы с прогнозами доходов от грузовых перевозок. Бэкенд с ML-моделями и расчетной логикой разрабатывается параллельно. Наша задача — создать **фронтенд-приложение из 5 ключевых страниц**, которое будет:

- Визуализировать результаты прогнозов
- Позволять управлять сценариями
- Предоставлять дашборды для топ-менеджеров

---

## Функциональные требования

1. **Загрузка данных** — импорт Excel-файлов с грузовой базой
2. **Конструктор сценариев** — создание и редактирование параметров прогноза
3. **Визуализация маршрутов** — отображение грузопотоков на карте/графе
4. **Дашборд доходов** — графики и таблицы с прогнозом на 10 лет
5. **Администрирование** — управление пользователями и история расчетов

## Структура приложения: 5 страниц

### 1. Страница загрузки данных (Data Import)
**Путь:** `/import`

**Назначение:** Загрузка исходных Excel-файлов от пользователя.

**Элементы:**
- Drag-and-drop зона загрузки
- Предпросмотр загруженных данных (таблица)
- Валидация структуры файла
- Прогресс загрузки
- Кнопка "Отправить на расчет"

### 2. Страница конструктора сценариев (Scenario Builder)
**Путь:** `/scenarios/new` и `/scenarios/:id/edit`

**Назначение:** Создание и настройка параметров прогноза.

**Элементы:**
- Форма с параметрами:
  - Выбор периода прогноза (2026-2036)
  - Ползунки изменения объемов грузов
  - Выбор направлений (откуда/куда)
  - Типы грузов (номенклатура)
  - Добавление новых клиентов
- Блок с текущими параметрами сценария
- Кнопки "Сохранить", "Рассчитать", "Сравнить"

### 3. Страница визуализации маршрутов (Route Visualization)
**Путь:** `/scenarios/:id/routes`

**Назначение:** Визуальное отображение грузопотоков на карте или графе.

**Элементы:**
- Интерактивная карта/граф ЖД-сети
- Отображение выбранных маршрутов
- Фильтры по типам грузов и направлениям
- Информация о маршруте при наведении
- Легенда (цветовая кодировка)

### 4. Страница дашборда доходов (Revenue Dashboard)
**Путь:** `/scenarios/:id/dashboard`

**Назначение:** Визуализация прогнозов доходов для топ-менеджеров.

**Элементы:**
- График прогноза на 10 лет (линейный/столбчатый)
- Сравнение нескольких сценариев (до 3)
- Ключевые показатели (карточки):
  - Общий доход за период
  - Среднегодовой рост
  - Доход по типам грузов
  - Доход по направлениям
- Выгрузка отчета (PDF/Excel)
- Фильтры по датам и категориям

### 5. Страница истории и администрирования (History & Admin)
**Путь:** `/history` и `/admin`

**Назначение:** Управление сохраненными сценариями и пользователями.

**Элементы:**
- Список сохраненных сценариев (таблица)
- Поиск и фильтрация по дате/названию
- Действия: просмотр, редактирование, удаление
- Сравнение выбранных сценариев
- (Для админа) Управление пользователями и правами доступа

## API интеграция

### Эндпоинты для фронтенда (предполагаемые):

| Метод | Эндпоинт | Назначение | Страница |
|-------|----------|------------|----------|
| `POST` | `/api/upload` | Загрузка Excel-файла | Импорт |
| `GET` | `/api/scenarios` | Получение списка сценариев | История |
| `POST` | `/api/scenarios` | Создание нового сценария | Конструктор |
| `GET` | `/api/scenarios/:id` | Получение сценария по ID | (все) |
| `PUT` | `/api/scenarios/:id` | Обновление сценария | Конструктор |
| `DELETE` | `/api/scenarios/:id` | Удаление сценария | История |
| `POST` | `/api/scenarios/:id/calculate` | Запуск расчета | Конструктор |
| `GET` | `/api/scenarios/:id/results` | Получение результатов | Дашборд |
| `GET` | `/api/scenarios/:id/routes` | Получение данных для маршрутов | Визуализация |
| `GET` | `/api/users` | Управление пользователями | Админка |
